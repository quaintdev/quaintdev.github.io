Part 1: Creating Todo Application With Htmx

August 19, 2023 · 3 min · Rohan D

This is beginner’s introduction to creating a web based todo application. If you program on daily basis this post is not for you. But before you go I want you to know that I am implementing this application in htmx. If you haven’t heard of htmx, read along. You may skip the sections depending on your expertise

Our todo application will be very basic one. It will support functionalities like

  • adding todo

  • listing previously added todos

  • allow marking as done to added todos

Let’s get started#

Our application will be a client/server application. We will be using HTMX for frontend, Go for backend and a Json data structure to persist the data. Let’s get started

The frontend#

Below is the core markup of our application. Go through it once before I explain what it means below

<ul>
    <div id="todo-placeholder"></div>
</ul>
<p><input type="text" placeholder="Enter a todo item here"
          hx-get="/todo/add" hx-target="#todo-placeholder"
          hx-swap="beforebegin" name="todoitem" /></p>

Here we are creating an empty, invisible<div> and a text box which will be used for entering todo items. The parameters like hx-get, hx-target are specific to htmx. When the user has updated the text box above snippet basically calls /todo/add endpoint on our server. The response of our server is added beforebegin of placeholder <div> as indicated by hx-target and hx-swap parameter.

The backend#

Let’s start by implementing first API endpoint /todo/add. Before implementing we need to register the endpoint with the Go server that our application will start.

http.HandleFunc("/todo/add", handleAdd)
http.ListenAndServe(":8080", nil)

On the first line we are registering /todo/add endpoint with server. http.HandleFunc is a library function of http package which takes API endpoint as first argument while a function as a second argument. Yes you can pass a function just like you would pass any other arguments. handleAdd is not just any other function it has special signature. We will learn more about it later. Basically the first line means execute handleAdd when user requests /todo/add. The second line is where we start our server on port 8080

Below is the most basic implementation of handleAdd. When the client calls /todo/add we basically provide a todo item with a checkbox. This function does not persist the data anywhere. It merely extracts the data from request, updates the markup with todo and sends the data.

func handleAdd(w http.ResponseWriter, r *http.Request) {
	id := RandStringRunes(3)
	item := r.URL.Query()["todoitem"][0]
	html := fmt.Sprintf(`
	<li>
		<input type="checkbox" id="%s">
    	<label for="%s">%s</label>
	</li>`, id, id, item)

	w.Write([]byte(html))
}

This concludes part 1 of our todo application. In next post we will dive deeper into htmx and persist the data to permanent storage. Meanwhile you can checkout the htmx ~ Documentation. It’s fairly easy to setup.
